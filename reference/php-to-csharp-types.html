Quick reference · Brownfield baseline

PHP → C#: types change what code means

C# types participate in compilation, overload selection, permitted operations, representation, and dispatch. They are not merely annotations on PHP-like values.

Compile-time type versus run-time type

object value = "invoice"; // compile-time: object; run-time: string
Console.WriteLine(value.ToString());

The declared type controls which members and implicit conversions are available and participates in overload resolution. The object’s run-time type controls virtual dispatch and run-time type tests.

Fast comparison

PHP instinct

A variable can hold values of unrelated types over its lifetime. Type declarations mainly constrain boundaries and assignments at run time.

C# model

Every expression has a compile-time type. A variable’s declared or inferred type does not change after declaration.

Three distinctions to inspect

  • Value/reference: assignment may copy a value or copy a reference.
  • Static/run-time: the same object can be viewed through different compile-time types.
  • Implicit/explicit conversion: a cast requests a defined conversion; it does not generally coerce arbitrary data.
Version note: These fundamentals are brownfield baseline. Newer syntax can express types differently without removing these semantics.
